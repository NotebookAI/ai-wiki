### 项目简介

在 AI 领域，名词爆炸式增长：从基础模型、训练范式，到 RAG、Agent、多模态工作流，很容易「听过很多词，但记不清含义和关系」。

这个项目想做的是一件简单但不容易的事：

- 把跑得很快的前沿名词，翻译成尽量清晰、克制的中文说明；
- 按主题和场景整理，帮你建立一张「AI 概念地图」；
- 通过社区协作，持续更新、修正、补充，让术语跟得上实践；
- 借助 GitHub 账号和内置评论区，让对同一术语感兴趣的人在同一页下相遇、讨论，并顺手互相「点进主页看看对方在做什么」。

它不是「权威标准」，而是一份开放的公共笔记本：任何人都可以提出不同理解、补充案例或更好的说法，一起把它打磨得更好懂；评论区里留下的交流和争论，也会慢慢成为这份笔记的一部分。

- 在你们内部文档、知识库或培训材料中引用本百科的链接，从同一套术语出发沟通，减少「同词不同意 / 同意不同词」；

## English

AI Glossary is an open, community-driven knowledge base for AI / LLM / RAG / Agent terminology. It is for anyone curious about how AI is evolving—not just engineers—and aims to explain “what on earth is actually happening” in this fast-moving space.

> You can treat it both as a “reference dictionary” when you run into an unfamiliar term and as a “navigation page” when you are trying to build a bigger-picture mental model.

### Project Overview

The AI landscape churns out new buzzwords daily: model architectures, training strategies, RAG pipelines, agents, multimodal workflows, and beyond. It is very easy to feel like you have “heard the word before” but cannot recall what it really means or how it connects to others.

This glossary tries to do a few simple things well:

- Offer clear, down-to-earth explanations instead of marketing speak;
- Organize terms by topic and use case so you can build a mental map;
- Continuously evolve through community contributions as practices change.

It is not meant to be a “final authority”, but an open notebook where people can bring in their understanding, references, and counterpoints—and gradually make the concepts easier to learn.

### Visit Online

- Live site: <https://notebookai.github.io/ai-wiki/>

You can:

- Browse through full-text search, category filters, or card/list view toggles;
- Dive into a term page for detailed explanations and references;
- Leave comments via giscus at the bottom of each term page.

### How to Use

**Readers / Learners**

- Search any keyword, alias, or English/Chinese name on the home page;
- Filter by categories like `foundation`, `rag`, `agent`, etc.;
- Open a card to read the full entry, examples, and related resources;
- Use the giscus comments to share your understanding, ask questions, or point out where the explanation could be clearer.

**Teams / Organizations**

- Treat this repo as the upstream “shared glossary” so your team can open PRs here and keep terminology aligned in one place;
- Reference this glossary from your internal docs, wikis, or training material so everyone talks about the same vocabulary;
- If you need a customized deployment or appearance, refer to GitHub Pages / Jekyll documentation and configure things as needed.

### Contributing

We welcome:

- New or improved term entries;
- Better taxonomy / keyword coverage;
- Styling or UX improvements.

Whether you are new to GitHub or already used to engineering workflows, as long as you want to help make concepts easier to understand, you are very welcome here.

Recommended contribution flow (no heavy setup required):

- Use the “Edit this term” / “Add term” buttons on the site, or edit/create Markdown files under `_terms/` via GitHub’s web UI;
- Add or adjust content based on the term template, and include a short note describing your changes when you submit;
- GitHub will walk you through creating a Pull Request, and we will review and merge it into the main site.

All improvements land in this single upstream glossary, which makes it easier to maintain and evolve over time and lets future readers benefit directly from your work.

For details about term templates, repository structure, local development, and giscus configuration, please refer to [DEV.md](DEV.md).
